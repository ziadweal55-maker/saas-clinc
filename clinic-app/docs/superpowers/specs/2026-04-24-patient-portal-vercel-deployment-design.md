# Design Doc: Patient Portal Vercel Deployment

**Date:** 2026-04-24
**Topic:** Deploying `patient-portal` to Vercel via CLI

## Overview
The goal is to move the Patient Portal from local hosting to a public URL on Vercel. This allows patients to access their exercise plans and progress from anywhere using the QR codes generated by the main Clinic Manager app.

## Architecture
- **Frontend:** React (Vite) app located in `/patient-portal`.
- **Hosting:** Vercel (Static Site Hosting).
- **Backend/Database:** Supabase (existing).
- **Deployment Method:** Vercel CLI (Manual push).

## Infrastructure & Configuration
### Environment Variables
The following variables must be configured in Vercel to allow the portal to communicate with the database:
- `VITE_SUPABASE_URL`: The Supabase project URL.
- `VITE_SUPABASE_ANON_KEY`: The Supabase anonymous API key.

### Vercel Project Settings
- **Root Directory:** `patient-portal`
- **Build Command:** `npm run build`
- **Output Directory:** `dist`
- **Install Command:** `npm install`

## Implementation Steps
1. **Prepare Environment:** Ensure Vercel CLI is installed and authenticated.
2. **Project Linking:** Link the `patient-portal` directory to a new Vercel project.
3. **Configure Secrets:** Set the Supabase environment variables in the Vercel dashboard or via CLI.
4. **Deploy:** Execute `vercel --prod` from the `patient-portal` directory.
5. **Verify:** Access the generated `.vercel.app` URL and confirm data is loading from Supabase.
6. **Update Electron App:** Update the QR code generation logic in the main app to use the new production URL.

## Success Criteria
- Patient Portal is reachable via a public URL.
- Patients can login with their PIN and see their synced data.
- The Electron app generates QR codes that point to the Vercel URL.
